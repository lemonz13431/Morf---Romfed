Lexicon categories

Morphemes live here: prefixes, roots, suffixes, stems, infixes, particles. Their letter codes work in generator patterns.

Vocabulary categories

Whole words live here. Use dot variables like .n., .verb., or .place. in generator patterns.

Name categories

Proper names live here: people, places, titles, groups, deities, and story-world names. Use double-dot variables like ..F.. or ..place.. in generator patterns.

Name entries use one line each: Name, Nickname = actual meaning. Variations like Jord[a/y]n and Carol[i/y]n(e) expand into spelling variants. Nickname groups can use brackets, like Elizabeth, [Liz/Lizzy/Elsa] = name meaning.

Additional patterns / graphemes

Reusable phoneme or grapheme variables. These can reference each other and lexicon categories.

Syntax help

This is the Morf Version 4 grammar in one place.

Basic pattern syntax

C V N
Uppercase letters expand additional patterns and/or Lexicon categories with that letter.
B = C/N/L/K or B = C, N, L, K
Nested variables keep resolving until they become real output; if K = s[t/n], then B can generate st or sn.
a/e/i
Slash alternatives. One branch is chosen.
[CV]
Required group; everything inside acts as one unit.
(CV)
Optional group with a 50% chance.
a*5/e/i
Weights. The weighted option is more likely.
"[literal]"
Quotes escape special syntax characters.

Morf additions

.n.
Choose a whole word from a Vocabulary category.
..F..
Choose a proper name from a Names category. Names can keep capitals and spaces.
<CV>&
Capture then repeat the last capture.
&1
Repeat the first capture.
C{3}, [CV]{2,4}
Exact and range repetition.
V!a,e
Exclude outputs from a variable.
[VV]^aa^ii
Awkwords-style local filtering.

Advanced rules

ti=chi
Rewrite generated text after building a word.
<C>=&1&1
Capture in rewrite targets, then reuse the captured text.
Cor=Nor
Variables in rewrites: generated consonant + or becomes generated nasal + or.
VVV
Forbidden sequence using variables.
<C>&1
Forbidden repeated captured consonant, useful for blocking geminates.
$pep$
In Starts with / Ends with only, try to fit that literal into the existing pattern; if it cannot fit, prepend or append it. Contains stays a normal filter.

Translator behavior

The analyzer builds a tile set from Lexicon morphemes, Vocabulary whole words, and Names. Exact vocabulary matches are preferred, but mixed morpheme parses are still listed as alternatives.

Placement rules are strict: start entries only parse in an unbroken start-zone, middle entries cannot parse at the beginning or end, and end entries only parse in an ending-zone.

Click generated or translator segments to add them. Click dictionary entries to edit them, or move them between Vocabulary, Lexicon, and Additional patterns.

Synonyms, homonyms, and variations

mor[o/u] = water or dar/dra = before
Slash/optional forms are variation words inside one entry. The dictionary shows them under “Show variation words.”
dar,dra = before
Commas on the form side create separate synonym entries with the same exact meaning.
Bri = from and Feez = from
Separate entries with the same exact meaning are also treated as synonyms.
kyme = magic(al) or co(ur)a = beach/sand
Optional/slash meanings create multiple meanings for the same word, shown as additional meanings.

Family / Related to links

peachy = peach-like ;{V{Nouns}peach};
Links this vocabulary entry to the vocabulary word peach in the Nouns category.
Dariaville = town ;{N{First names}Daria};
Links a name to another name. The entry shows “Related to”; the target shows “See family.”
joopish = bread-like ;{L{R}joop};
Links to a Lexicon entry. Use L, V, or N for Lexicon, Vocabulary, or Names.
moru~ / kime~
Single tilde expands the stored spelling variations for the entry containing that form. If mor[o/u] exists, moru~ can mean moro or moru.
peach~~
Double tilde expands the broader family around that form: the entry itself, entries it is related to, and entries that point to the same family target.
